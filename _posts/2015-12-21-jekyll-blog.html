In this tutorial, I will show you how to get up and running with a basic Jekyll blog. At the end of this tutorial is a section on issues I had along the way/things I learned. You might also find it helpful to take a sneak peek at that first!

To install Jekyll, follow the steps below:

  1. This installs jekyll

  2. This creates a new jekyll project (make sure you go into your project before the next step)

  3. This starts up the jekyll-provided local server

  4. Go to the config.yml file to update it with your blog's information (Tip: Every time you change the config file, save it, exit the server by hitting Control + C, type "jeckyll serve" again, and then refresh the page.)

  5. Go to Jekyll's "about.md" page and update it with information about yourself (Note: you do not need to restart the server if you make changes outside of the config file- just refresh the browser after changes).

  6. As default, the Jekyll url is written in this format: category/year/month/daye/title and ends in .html. If you add “permalink: pretty” to your config under Build Settings, it removes the .html. If you set “permalink: /:title”, your blog will only contain the title.

  7. Go to Jekyll's "_posts" directory and use its sample welcome file to create your first blog.

    In the example post that Jekyll created, you will see a block of text at the top sandwiched inside triple dashed lines. This is called a YAML front matter block and Jekyll uses them to maintain a page and its contents.

    Modify the predefined variables to fit your first blog. Keep layout as post. Change the title to the title of your blog post. Update the categories words to define your blog, each word separated by a space. Change the content of your post. Add author as a variable to the YAML front matter block if you'd like to access it for your posts.

  8. The url is not updating with the correct title. That’s because it is using the filename of the blog post in order to update the url. Instead of updating the url based on the file name, we can set a permalink variable in that blog’s front line matters block to set the url to what we would like it to be.

    Note: You will have to go back to the main Jekyll page, refresh the page, and go back into the blog post to see the change to the url.

  9. Notice below in the index.html how the use of "site.posts" prints out each of your posts. For each post, you have access to certain information that you can use wherever you want on that post's page.

    • post.title
    • post.date
    • post.author
    • post.excerpt

  10. You can use custom variables in your front matter block such as "image." Set the image to what you want it to be in your front matter block.

  14. To add new posts, simply add a file in the "_posts" directory that follows the convention "YYYY-MM-DD-name-of-post.ext" and includes the necessary front matter.

  15. What if you want to work on a post but not publish it yet? To create a draft, 1) Make a new directory called _drafts. 2) Create a regular post in this directory (the date is optional).

  16. To see this draft show up in your index, restart the server using the command "jekyll serve drafts"

There you go!

Check out your new blog! Ok, so it's not the most amazing blog ever, but it's a good starting point. Here are a few issues I ran into/things I learned along the way:

  • You do not have to use markdown for the markup (whaaaa???). I changed the markdown extensions for my blog posts to html because I am more comfortable with html.
  • I had to change the way I referenced images once I deployed to Github Pages (duh!). Instead of using relative paths that pointed to the image folder I created, I had to use site.baseurl.

    You will need to make this change for the image variables in your front matter as well as other images in your blog post.
